## Remaining JIT Parity Gaps

- [ ] 3+ level nested lambda captures not resolved in JIT flat literal pass
  - Affects: `examples/basics/list_comprehension.flx` (Pythagorean triples)
  - Root cause: `compile_ir_literal_functions` compiles all literals in flat pass;
    transitive captures through 3+ levels don't resolve from parent scope
  - Workaround: 2-level nesting works; restructure code to avoid 3+ levels

- [ ] VM `print` uses `Display` (adds quotes around strings), JIT does not
  - Affects: `match_wildcard_non_last_error.flx`, `match_non_exhaustive_error.flx`
  - Root cause: VM `OpToString`/`print` uses `Value::Display` (`"hello"`),
    JIT uses `to_string_value()` (`hello`)
  - Fix: unify print formatting between VM and JIT

- [ ] Pattern validation complex match arms
  - Affects: `examples/patterns/pattern_validation_happy_path.flx`
  - Output content matches but formatting differs

- [ ] Higher-order effectful functions in JIT CFG
  - Affects: 5 type_system effect-row examples (100-104)
  - Root cause: CFG tail-call targets for function parameters not resolved

- [ ] Public HOF contract in JIT
  - Affects: `examples/type_system/204_strict_public_hof_contract_ok.flx`
  - VM succeeds, JIT fails
